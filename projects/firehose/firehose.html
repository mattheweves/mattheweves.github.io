The Firehose Project


I decided to enroll in the Firehose Project after many years of being in a corporate environment where I felt like learning had plateaud (at least from a technical perspective) and I was hungry for challenge.

I did a full year of CS in college and had already compiled and written applications in C++ and Javascript, but it had been 7 years since that time and felt a lot of what I had learned was "dated" (although I now see those classes and introduction to software engineering as invaluable). I felt it nearly impossible to keep up without a supportive community and a curriculum that would at least bring me up to speed on the modern frameworks used and coding fundamentals.

The Firehose Project owns up to its name (drinking from the firehose) and reputation as a top notch software engineering immersive that delivered above and beyond in every respect. Their staff and mentors were incredibly bright and helpful and it confirmed a thousand questions about what it is like to be a modern day developer. We built 4 web applications in increasing difficulty and at the end worked as a team to build an online chess app. The goal of this was to actually work on a team, experience first hand git workflows and see how sprints are managed (we used Trello).

About a year later, I also signed up and completed the Advanced Track which included crash courses in React, SQL Databases: N+1 Queries, Complex Joins, SQL Injection, etc. and RESTful API Design. After all of this, I tried to put a lot of what I had learned together in my own React JS Web App, KIDFO using a RESTful API built on Rails 5 and React JS for the front end.

Some Project Apps



Random Quote Generator
The first was a Rails App with a very simple database structure, just one table that holds quotes and randomly displays them on the home page. I chose the uplifting topic of "Extinction" for mine.

People can add quotes to the database via a simple form contained in a bootstrap modal. Some basic data validation is also performed to ensure they fill out the author and quote.

Concepts introduced: MVC Framework, Sample Controller/Routes, Database creation/migration/initialization, Accessing Data using Ruby, CSS, Git Tracking, Deployment to Heroku, Simple Forms, Gem Installation

Picnic App
A more complicated Rails Application that introduced account creation (Devise), models, working with both a Geocoder API and Google Maps to store and display locations, and more object relationships (comments, ratings).

Concepts introduced: Devise and User Authentication, More advanced Models, Geocoder API for translating and address to Latitude/Longitude, Google Maps Integration, and Object Relationships (example: comments belong to Place)





React JS Note App with Rails API (backend)
First app using "microservices". Simple and great way to jump in to React is the classic "Note App".

Concepts introduced: React, Single Page Application (SPA), State
